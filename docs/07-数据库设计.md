# 07 - 数据库设计文档

## 1. 数据库概述

- **数据库类型**: SQLite
- **ORM/驱动**: better-sqlite3
- **文件位置**: `backend/database.db`
- **字符编码**: UTF-8

---

## 2. 核心表设计

### 2.1 用户表 (users)

```sql
CREATE TABLE users (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  username TEXT UNIQUE NOT NULL,          -- 用户名（唯一）
  password TEXT NOT NULL,                 -- 密码（bcrypt加密）
  real_name TEXT,                         -- 真实姓名
  phone TEXT,                             -- 手机号
  email TEXT,                             -- 邮箱
  status TEXT DEFAULT 'active',           -- 状态: active, inactive
  department_id INTEGER,                  -- 部门ID
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  last_login_at DATETIME,
  
  FOREIGN KEY (department_id) REFERENCES departments(id)
);

-- 索引
CREATE INDEX idx_users_username ON users(username);
CREATE INDEX idx_users_status ON users(status);
CREATE INDEX idx_users_department ON users(department_id);
```

**字段说明:**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | INTEGER | 自动 | 主键 |
| username | TEXT | 是 | 用户名，唯一 |
| password | TEXT | 是 | bcrypt加密密码 |
| real_name | TEXT | 否 | 真实姓名 |
| phone | TEXT | 否 | 手机号 |
| email | TEXT | 否 | 邮箱 |
| status | TEXT | 否 | 状态：active/inactive |
| department_id | INTEGER | 否 | 部门ID |
| last_login_at | DATETIME | 否 | 最后登录时间 |

### 2.2 角色表 (roles)

```sql
CREATE TABLE roles (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  name TEXT NOT NULL,                     -- 角色名称（显示名）
  code TEXT UNIQUE NOT NULL,              -- 角色代码（唯一）
  description TEXT,                       -- 角色描述
  permissions TEXT,                       -- 权限列表（JSON数组）
  status TEXT DEFAULT 'active',           -- 状态
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 索引
CREATE INDEX idx_roles_code ON roles(code);
CREATE INDEX idx_roles_status ON roles(status);
```

**permissions字段示例:**
```json
[
  "project:view",
  "project:create",
  "project:edit",
  "contract:view",
  "contract:approve"
]
```

### 2.3 用户角色关联表 (user_roles)

```sql
CREATE TABLE user_roles (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  user_id INTEGER NOT NULL,
  role_id INTEGER NOT NULL,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  
  FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,

);
```

### 2.4 部门表 (departments)

```sql
CREATE TABLE departments (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  name TEXT NOT NULL,                     -- 部门名称
  parent_id INTEGER,                      -- 父部门ID
  manager_id INTEGER,                     -- 部门负责人ID
  level INTEGER DEFAULT 1,                -- 层级
```

### 2.6 合同表 (contracts)

```sql
CREATE TABLE contracts (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  contract_no TEXT UNIQUE NOT NULL,       -- 合同编号
  contract_type TEXT,                     -- 类型: income, expense
  project_id INTEGER,                     -- 项目ID
  supplier_id INTEGER,                    -- 供应商ID
  party_a TEXT,                           -- 甲方
  party_b TEXT,                           -- 乙方
  contract_category TEXT,                 -- 合同类别: equipment, material, labor
  total_amount DECIMAL(15,2),             -- 总金额
  paid_amount DECIMAL(15,2) DEFAULT 0,    -- 已付款金额
  sign_date DATE,                         -- 签订日期
  start_date DATE,                        -- 开始日期
  end_date DATE,                          -- 结束日期
  payment_terms TEXT,                     -- 付款条款
  status TEXT DEFAULT 'draft',            -- 状态
  remark TEXT,                            -- 备注
  creator_id INTEGER,                     -- 创建人ID
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  
  FOREIGN KEY (project_id) REFERENCES projects(id),
  FOREIGN KEY (supplier_id) REFERENCES suppliers(id),
  FOREIGN KEY (creator_id) REFERENCES users(id)
);

-- 索引
CREATE INDEX idx_contracts_no ON contracts(contract_no);
CREATE INDEX idx_contracts_type ON contracts(contract_type);
CREATE INDEX idx_contracts_project ON contracts(project_id);
CREATE INDEX idx_contracts_status ON contracts(status);
```

## 9. 索引策略

### 9.1 主键索引

- `roles.code`
- `projects.project_no`
- `contracts.contract_no`

- `purchase_lists.list_no`

### 9.5 时间索引
常用的时间字段（如 `created_at`）在需要按时间范围查询时应创建索引

## 11. 备份策略

### 11.1 自动备份脚本

```bash
#!/bin/bash
# backup.sh

DATE=$(date +%Y%m%d_%H%M%S)
BACKUP_DIR="/backups"
DB_FILE="backend/database.db"

# 创建备份目录
mkdir -p $BACKUP_DIR

# 备份数据库
cp $DB_FILE "$BACKUP_DIR/database_$DATE.db"

# 压缩备份
gzip "$BACKUP_DIR/database_$DATE.db"

# 删除7天前的备份
find $BACKUP_DIR -name "*.gz" -mtime +7 -delete

echo "Backup completed: database_$DATE.db.gz"
```

## 12. 性能优化

### 12.1 查询优化

- 使用 `EXPLAIN QUERY PLAN` 分析查询
- 避免 `SELECT *`，只查询需要的字段
- 合理使用索引
- 避免 N+1 查询问题

### 12.2 事务处理

```javascript
// 使用事务批量插入
const insert = db.transaction((items) => {
  for (const item of items) {
    db.prepare('INSERT INTO table VALUES (?, ?)').run(item.a, item.b);
  }
});

insert(items);
```

### 12.3 连接池

SQLite是嵌入式数据库，不需要连接池，但better-sqlite3会自动管理连接。
