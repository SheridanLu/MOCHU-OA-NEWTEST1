## 一、项目概述

### 1.1 项目背景

MOCHU-OA是一套面向施工企业的综合办公自动化系统，旨在解决施工企业在项目管理、合同管理、采购管理、库存管理、财务管理等方面的信息化需求。

### 1.2 系统目标

- 实现施工项目全生命周期管理
- 规范合同审批和采购流程
- 精细化库存和成本控制
- 多角色协同办公

### 1.3 目标用户

| 角色 | 职责 |
|------|------|
| 总经理 | 全局管理、最终审批 |
| 财务管理 | 财务审批、费用控制 |
| 项目经理 | 项目管理、进度把控 |
| 采购员 | 采购执行、供应商对接 |
| 预算员 | 成本预算、超量审核 |
| 法务 | 合同审核、风险控制 |
| 资料员 | 文档归档、图纸管理 |
| 人事 | 人员管理、考勤管理 |

## 二、项目管理模块

### 2.1 项目类型

系统支持两种项目类型：

#### 2.1.1 虚拟项目
- **定义**: 尚未中标的项目，用于前期成本跟踪
- **编号规则**: 8位数字
- **状态**: 跟踪中、已转换、已中止

#### 2.1.2 实体项目
- **定义**: 已中标并签订合同的项目
- **编号规则**: 10位数字
- **状态**: 进行中、已完成、已暂停

### 2.2 项目立项流程

```
采购员提交立项申请 → 财务审批 → 总经理审批 → 立项完成
```

**立项信息包含**:
- 项目基本信息（名称、类型、金额）
- 项目经理指派
- 计划开始/结束日期
- 项目分类（智能化/消防/EPC等）

### 2.3 虚拟项目转实体项目

**触发条件**: 项目中标

**操作流程**:
1. 采购员上传中标通知书
2. 填写中标日期、中标金额
3. 提交转换审批（采购员→财务→总经理）
4. 审批通过后自动生成实体项目编号
5. 虚拟项目下所有文件自动归集到实体项目

### 2.4 虚拟项目中止

**触发条件**: 项目未中标或放弃

**操作流程**:
1. 采购员填写中止原因
2. 选择成本下挂目标（实体项目或部门成本）
3. 提交中止审批（采购员→财务→总经理）
4. 审批通过后：
   - 项目状态变更为"已中止"
   - 页面冻结，不可继续操作
   - 所有成本归集到指定目标

### 2.5 项目分类

| 分类 | 说明 |
|------|------|
| 智能化项目 | 智能化系统集成项目 |
| 消防项目 | 消防系统工程项目 |
| EPC项目 | 设计采购施工总承包项目 |
| 其他 | 其他类型项目 |

## 三、合同管理模块

### 3.1 合同类型

#### 3.1.1 收入合同
- 与客户签订的合同
- 关联项目信息
- 支持合同金额拆分

#### 3.1.2 支出合同
- 与供应商/分包商签订的合同
- 分为：设备类、材料类、劳务类、施工类

### 3.2 支出合同字段要求

**基本信息**:
- 合同编号（自动生成）
- 合同名称
- 关联项目
- 供应商信息
- 合同金额
- 签订日期
- 合同期限

**扩展信息（物流与验收）**:
- 运输方式（甲方负责/乙方负责）
- 接货人
- 交货正负差
- 交货期限
- 货款结算方式
- 验收负责人
- 质保期

### 3.3 合同审批流程

```
提交合同 → 财务审批 → 法务审批（可选）→ 总经理审批 → 合同生效
```

### 3.4 合同超量校验

**触发条件**: 采购明细金额超过合同总价的1.5%

**处理流程**:
1. 系统自动标记超量
2. 需要提交超量审批
3. 审批流程：预算员→总经理
4. 审批通过后才能继续

### 3.5 合同模板管理

**功能需求**:
- 支持创建收入/支出合同模板
- 模板分类：设备类、材料类、劳务类、施工类
- 模板变量自动替换
- 支持设置默认模板

**常用变量**:
- 合同编号、合同名称
- 甲方、乙方
- 合同金额、金额大写
- 签订日期、开始日期、结束日期
- 项目名称、项目编号

## 四、采购管理模块

### 4.1 采购清单

**清单分类**:
- 设备类：名称、单位、数量、价格
- 材料类：名称、规格型号、单位、数量、价格

**清单状态**:
- 草稿、待审批、已审批、已完成

### 4.2 批量采购

**触发条件**: 多个采购清单汇总达到批量采购阈值

**功能**:
- 自动汇总同类型采购需求
- 生成批量采购订单
- 价格预警（超过基准价10%）
- 审批流程：采购员→财务→总经理

### 4.3 零星采购

**适用场景**: 紧急小额采购，金额不超过收入合同的0.5%

**信息要求**:
- 采购名称
- 关联收入合同
- 税票类型（无/普票/专票）
- 税率（0%/1%/3%/6%/9%/13%）
- 采购明细

**审批流程**:
- 普通采购：财务→总经理
- 超限采购：预算员→财务→总经理

## 五、库存管理模块

### 5.1 物资入库

**入库流程**:
1. 创建入库单（草稿）
2. 添加入库明细

**入库信息**:
- 入库单号
- 关联项目
- 供应商
- 入库日期

**领用流程**:
1. 选择领用项目
2. 选择领用物资
3. 填写领用原因
4. 提交审批

**审批流程**:
- 资料员→采购管理→总经理

### 5.3 库存查询

**查询维度**:
- 物资种类统计
- 总库存金额
- 预警物资（低于预警值）
- 紧急物资（低于最低值）
- 超储物资（超过最大值）
