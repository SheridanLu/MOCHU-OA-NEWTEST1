# 01 - 系统架构

## 1. 系统概述

MOCHU-OA采用前后端分离的架构设计，前端使用React构建单页应用(SPA)，后端使用Node.js + Express提供RESTful API服务。

### 4.3 Axios配置 (services/api.js)

```javascript
import axios from 'axios';

const api = axios.create({
  baseURL: window.location.origin + '/api',
  timeout: 10000,
  headers: {
    'Content-Type': 'application/json'
  }
});

// 请求拦截器 - 添加token
api.interceptors.request.use(
  (config) => {
    const token = localStorage.getItem('token');
    if (token) {
      config.headers.Authorization = `Bearer ${token}`;
    }
    return config;
  },
  (error) => Promise.reject(error)
);

// 响应拦截器 - 处理滑动过期
api.interceptors.response.use(
  (response) => {
    // 检查是否有新token（滑动过期自动刷新）
    const newToken = response.headers['x-new-token'];
    if (newToken) {
      localStorage.setItem('token', newToken);
    }
    return response;
  },
  (error) => {
    if (error.response?.status === 401) {
      localStorage.removeItem('token');
      localStorage.removeItem('user');
      window.location.href = '/login';
    }
    return Promise.reject(error);
  }
);

export default api;
```

## 6. 认证流程

### 6.1 登录流程

```
用户输入账号密码
        │
        ▼
POST /api/auth/login-password
        │
        ▼
验证账号密码
        │
        ├── 失败 → 返回错误信息
        │
        └── 成功
            │
            ▼
    生成JWT Token (有效期30天)
            │
            ▼
    返回 { token, user }
            │
            ▼
    前端存储到localStorage
            │
            ▼
    跳转到首页
```

### 6.2 Token验证流程

```
前端发起请求
    │
    ├─ 添加 Authorization: Bearer <token>
    │
    ▼
后端authMiddleware
    │
    ├─ 验证token
    │   │
    │   ├── 无效 → 401 Unauthorized
    │   │
    │   └── 有效
    │       │
    │       ├─ 解析用户信息
    │       │
    │       ├─ 挂载到req.user
    │       │
    │       └─ 检查是否需要刷新
    │           │
    │           └─ 剩余时间<1小时 → 生成新token
    │               │
    │               └─ 响应头返回 X-New-Token
    │
    ▼
继续处理请求
    │
    ▼
前端检查响应头
    │
    └─ 有新token → 更新localStorage
```

### 6.3 滑动过期机制

```javascript
// 后端中间件 - 滑动过期
function authMiddleware(req, res, next) {
  // ... 验证token
  
  // 滑动过期：如果token即将过期（剩余时间<1小时），自动刷新
  const decodedExp = jwt.decode(token);
  const remainingTime = decodedExp.exp * 1000 - Date.now();
  const oneHour = 60 * 60 * 1000;
  
  if (remainingTime < oneHour && remainingTime > 0) {
    const newToken = generateToken(req.user);
    res.setHeader('X-New-Token', newToken);
  }
  
  next();
}

// 前端拦截器 - 自动更新token
api.interceptors.response.use((response) => {
  const newToken = response.headers['x-new-token'];
  if (newToken) {
    localStorage.setItem('token', newToken);
  }
  return response;
});
```

### 7.2 权限检查中间件

```javascript
function checkPermission(permission) {
  return (req, res, next) => {
    const userPermissions = getUserPermissions(req.user.id);
    
    // 超级管理员(GM)拥有所有权限
    if (userPermissions.includes('*')) {
      return next();
    }
    
    if (!userPermissions.includes(permission)) {
      return res.status(403).json({
        success: false,
        message: '无权限执行此操作'
      });
    }
    
    next();
  };
}

// 使用示例
router.post('/projects', 
  authMiddleware, 
  checkPermission('project:create'), 
  createProject
);
```

### 8.2 审批流程设计

```javascript
// 审批流程配置
const APPROVAL_FLOWS = {
  purchase_list: {
    steps: [
      { role: 'FINANCE', name: '财务管理' },
      { role: 'GM', name: '总经理' }
    ]
  },
  contract: {
    steps: [
      { role: 'FINANCE', name: '财务管理' },
      { role: 'LEGAL', name: '法务' },
      { role: 'GM', name: '总经理' }
    ]
  }
};

// 审批处理
async function approveApproval(approvalId, userId, comment) {
  // 1. 获取审批记录
  const approval = getApprovalById(approvalId);
  
  // 2. 检查当前用户是否有审批权限
  if (!canUserApprove(approval, userId)) {
    throw new Error('无审批权限');
  }
  
  // 3. 记录审批历史
  recordApprovalHistory(approvalId, userId, 'approved', comment);
  
  // 4. 更新审批状态
  if (isLastStep(approval)) {
    // 审批完成
    completeApproval(approvalId);
  } else {
    // 流转到下一审批人
    moveToNextStep(approvalId);
  }
}
```
